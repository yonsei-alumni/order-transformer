# 변환 규칙

## 구매자 주문 → 출고 지시 컬럼 매핑

| 구매자 주문 (원본) | 출고 지시 (변환 결과) | 변환 방식 |
|---|---|---|
| 주문번호 (B열) | 주문번호 (A열) | 그대로 복사 |
| 수령자명 (Y열) | 수령인 (B열) | 그대로 복사 |
| 주소 (AC열) + 상세주소 (AD열) | 수령인 주소(전체) (C열) | 공백으로 연결 |
| 배송지 우편번호 (AB열) | 수령인 우편번호 (D열) | 그대로 복사 |
| 수령자 전화번호 (Z열) | 수령인 휴대전화 (E열) | 그대로 복사 |
| (없음) | 자체품목코드 (F열) | 빈 값 |
| 구매수량 (Q열) | 수량 (G열) | 그대로 복사 |
| 배송메모 (AE열) | 배송메시지 (H열) | 그대로 복사 |
| 상품명 (R열) + 옵션명 (S열) | 주문상품명(옵션포함) (I열) | " / "로 연결 |

## 제외 조건

- **취소사유** (AG열)가 비어있지 않은 주문은 변환 대상에서 제외됩니다.

## 주소 결합 규칙

- 상세주소가 있는 경우: `주소 + " " + 상세주소`
- 상세주소가 없는 경우: `주소`만 사용

## 상품명 결합 규칙

- 옵션명이 있는 경우: `상품명 + " / " + 옵션명`
- 옵션명이 없는 경우: `상품명`만 사용

## 출력 파일

- 파일명: `{원본파일명}_출고지시_{yyyyMMdd}.xlsx`
- 시트명: `이지어드민 양식`
- 동일 파일명이 존재하면 `(1)`, `(2)` 등을 붙여 중복 방지
